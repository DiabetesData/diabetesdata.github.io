Supported Data Formats

Dexcom Clarity Export

Standard Dexcom Clarity CSV exports are supported.

The app uses the glucose timestamp and glucose value columns from the export, and reads device information when present.

LibreView Export

Standard LibreView CSV exports are supported.

The app reads device timestamp data and glucose values from the historic glucose and scan glucose fields when those rows are present.

Multi-Series CSV

A flexible CSV format is also supported when it includes these required columns:

Required: series, datetime_local, value

Typical series values: CGM for glucose rows, values containing insulin for insulin rows, meal announcement for meal markers, and event for pump or system events.

Optional columns: annotation_kind, tooltip_text, annotation_text_clean, annotation_text_raw, meal_type, meal_size_descriptor, event_type.

This format can be used either as a combined glucose plus insulin file or, after a CGM file is loaded, as an insulin-only supplemental file.

Simple Glucose CSV

A simple three-column glucose CSV is supported with these columns:

Expected columns: patient_id, blood_sugar, measurement_time

The app uses blood_sugar as the glucose value and measurement_time as the timestamp.

Times should be parseable as regular date-time strings. Files that include glucose values are automatically binned onto the app's analysis timeline for plotting and downstream calculations.
